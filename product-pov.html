Product Point of View

Quarterstack

A quarterly planning tool that replaces the spreadsheet-meeting-roadmap cycle with one continuous surface powered by AI agents.

The Problem

PMs don't lack frameworks. Every PM knows RICE, ICE, MoSCoW. The actual failure mode in quarterly planning is context switching. You pull data from Amplitude, customer feedback from Slack, revenue signal from Sales, ticket counts from Support, then consolidate it all into a spreadsheet, apply a framework, debate trade-offs in a meeting, and produce a plan in a third tool. Each switch is cognitive load. By the time you have a plan, it's already stale.

The Bet

Collapse the entire workflow into one continuous surface where scoring, deciding, and planning happen together, not sequentially.

Most tools separate "evaluate" from "decide" from "plan." You score features in one view, build a roadmap in another, and discuss trade-offs in a meeting. Quarterstack treats these as one fluid motion: features are pre-scored, the plan builds itself as you make decisions, and AI agents handle the grunt work of data collation and conflict detection. The PM's only job is judgment.

Key Design Decisions

1

No framework selection step

We auto-apply RICE and show a "How scores work" tooltip. No framework picker.

Why: Asking a PM to choose between RICE, ICE, and MoSCoW before they've even started is an analytical choice that adds friction with zero value. The framework is infrastructure. It should be invisible.

2

One scrollable surface, not tabs or steps

Backlog, cut line, and Gantt chart live on the same page. Scroll down = move through the workflow.

Why: Tabs and wizards create context switches. A PM adjusting a score needs to immediately see how it affects the plan. With one surface, cause and effect are visible simultaneously.

3

The Cut Line as the hero interaction

A draggable divider separates "this quarter" from "not this quarter." Everything above auto-populates the Gantt.

Why: The single most important decision a PM makes is: what's in, what's out. Not scores. Not frameworks. The cut. We made this the central, most tactile interaction. The plan IS the list above the line.

4

Two specialized AI agents, not one generic chatbot

An Analyst agent handles scores and data. A PO agent handles trade-offs and capacity. User switches via tabs.

Why: A generic "AI assistant" is a blank canvas. Specialized agents have clear jobs. This maps to how real teams work and aligns with the "Agents as Peers" principle.

5

Gantt with parallel streams, not monthly columns

The plan renders as a Gantt with 2 parallel work streams across 12 weeks, bars sized by effort.

Why: Monthly columns are a calendar, not a plan. The Gantt shows reality: parallelism, effort consumption, and capacity. A PM glancing at it knows instantly if the plan is feasible.

6

Pre-enriched features with source links

Every card arrives with scores pre-filled, data citations, and clickable source links (Slack, dashboards, meeting notes).

Why: The biggest time sink isn't scoring, it's gathering context. The Analyst agent has already done this. The PM reviews and adjusts, rather than starting from scratch.

What We Excluded

×

Team collaboration / comments

This is a single-PM planning tool. You discuss with the AI agents, not teammates in this flow.

×

Multi-quarter comparison

Scope creep. One quarter at a time. Comparing Q2 vs Q3 is a reporting tool, not a planning tool.

×

Custom framework builder

Serves 5% of PMs and violates zero cognitive load. RICE with adjustable sliders covers it.

×

Sprint-level capacity planning

The unit here is weeks, not story points. The Gantt gives "roughly feasible" which is what a quarterly plan needs.

×

Notifications / reminders

This is a "sit down and plan" experience, not a "check in daily" dashboard.

Design for Delight Alignment

ZCL

Zero Cognitive Load

No framework picker. Scores pre-filled. Plan auto-generates. The PM makes judgment calls, not data entry.

SD

Smart Defaults

Cut line starts at a sensible position. Scores pre-filled. New features get default scores. Dropdowns pre-selected.

PD

Progressive Disclosure

Collapsed cards show title + score. Expand for data, sources, sliders. Gantt appears below the backlog, visible when you're ready.

IF

Instant Feedback

Drag cut line, plan updates instantly. Adjust slider, score pulses and list reorders. Delete a card, slide-out animation + undo toast. Confetti when plan fits capacity.

PB

Purpose-built Design

The cut line is not a standard UI pattern. It's designed for the "what's in / what's out" decision. More intuitive than checkboxes or drag-to-bucket.

FM

Familiar Mental Models

Drag to reorder (Trello). Sliders for scoring (any form). Chat sidebar (ChatGPT). Gantt chart (every PM tool). Nothing requires learning.

If I Had More Time

1

Agent actions that modify the board

Right now agents give advice. Next step: "Move all infra features to Month 2" in the chat, and it actually happens. The PO agent becomes a pair-planning partner, not just an advisor.

2

Dependency visualization

Draw lines between features that depend on each other. The Gantt auto-sequences them so dependent features never start before their blockers. The PO agent flags circular dependencies.

3

Domain-specific agents: Design, Engineering, QA

Right now we have Analyst and PO. The next step is adding a Design agent that flags UX complexity, an Engineering agent that gives more accurate effort estimates based on codebase context, and a QA agent that identifies testing risks. Each agent brings domain expertise that a PM would normally get from separate meetings. The Gantt becomes more accurate because effort estimates come from the Engineering agent, not a generic 1-10 slider.

4

Scenario comparison

"What if we cut Custom Dashboards and add Slack integration instead?" Save two plan versions side-by-side. Compare effort, theme coverage, and risk profile. Make the trade-off visual, not verbal.
